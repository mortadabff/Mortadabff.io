Deep Learning et Machine Learning en géomatique

30 Mars 2024 Temps de lecture : 9 min

La géomatique vit une révolution silencieuse : là où on utilisait des seuillages et des indices spectraux, on déploie aujourd'hui des réseaux de neurones profonds et des forêts aléatoires. Cette transition méthodologique ouvre des horizons inédits pour l'analyse des territoires.

Mais face à la diversité des algorithmes (régression logistique, SVM, Random Forest, CNN, U-Net, Transformers), comment choisir ? Quand privilégier le machine learning "classique" et quand passer au deep learning ? Cet article vous guide à travers les possibilités, les contraintes et les bonnes pratiques spécifiques au domaine spatial.

1. Machine Learning vs Deep Learning : quel choix pour la géomatique ?

Machine Learning

  • Données : Petits à moyens volumes (ex: quelques centaines d'échantillons)
  • Features : Ingénierie manuelle (indices spectraux, textures)
  • Interprétabilité : Bonne (importance des variables)
  • Calcul : CPU suffisant, entraînement rapide
  • Exemples : Random Forest, SVM, XGBoost

Deep Learning

  • Données : Grands volumes nécessaires (milliers d'échantillons)
  • Features : Apprentissage automatique des représentations
  • Interprétabilit : Faible (boîte noire)
  • Calcul : GPU indispensable, temps long
  • Exemples : CNN, U-Net, Transformers

2. Applications en géomatique

Classification

Occupation du sol, typologie de cultures, zones à risque

ML: Random Forest DL: CNN

Détection d'objets

Bâtiments, véhicules, navires, panneaux solaires

DL: YOLO, Faster R-CNN

Estimation continue

Biomasse, température, concentration polluants

ML: XGBoost DL: MLP

Segmentation sémantique

Cartographie pixel-level, réseaux routiers

DL: U-Net, DeepLab

Interpolation spatiale

Krigeage, prédiction spatiale continue

ML: Krigeage, RFsp

Séries temporelles

Détection changements, phénologie

DL: LSTM, Transformers

3. Algorithmes incontournables

Machine Learning

Random Forest RF

Robuste, peu de paramètres, gère bien les données hétérogènes. Idéal pour classification d'occupation sol.

SVM SVM

Efficace en haute dimension, bon pour petits jeux de données. Sensible aux paramètres.

XGBoost GBM

Excellent pour régression et classification, souvent winner en compétitions.

Krigeage Géo

Méthode géostatistique de référence pour interpolation spatiale.

Deep Learning

CNN Conv

Base de la vision par ordinateur. Classification d'images satellite par patches.

U-Net Seg

Architecture encoder-decoder avec skip connections. Référence en segmentation.

DeepLab ASPP

Convolutions dilatées pour capturer contextes multi-échelles.

LSTM / Transformers Temp

Pour séries temporelles d'images satellite (ex: Sentinel-2 time series).

5. Bonnes pratiques pour données spatiales

Prétraitement adapté

Normalisation par capteur, gestion des ombres et nuages, indices spectraux (NDVI, MNDWI).

Augmentation de données

Rotations, flips, zoom, variations colorimétriques pour images satellite.

Validation spatiale

Cross-validation par blocs spatiaux pour éviter fuite de données.

Équilibrage classes

Classes déséquilibrées en occupation sol (ex: eau minoritaire).

Cross-validation spatiale : pourquoi c'est crucial

La validation classique par échantillonnage aléatoire est biaisée pour les données spatiales à cause de l'autocorrélation spatiale. Des points proches dans le train et le test surestiment les performances. Solution : découper l'espace en blocs indépendants.

6. Comment choisir ?

Critère ML classique Deep Learning

Type de données Tabulaire, indices, features Images, séries temporelles

8. Erreurs fréquentes

Validation aléatoire

Utiliser une validation aléatoire sur données spatiales donne des performances trop optimistes.

Ignorer les nuages

Ne pas masquer les nuages en imagerie satellite fausse les modèles.

Saisonnalité ignorée

Les modèles doivent intégrer la dimension temporelle (dates de prise de vue).

Généralisation spatiale

Un modèle entraîné sur une région ne fonctionne pas forcément ailleurs.

Conclusion

Le choix entre Machine Learning et Deep Learning en géomatique n'est pas idéologique mais pragmatique. Il dépend de vos données, de vos objectifs et de vos contraintes. L'essentiel est de maîtriser les spécificités spatiales : validation adaptée, prétraitements géographiques, et interprétation contextuelle des résultats.

Règle d'or : Commencez simple (Random Forest), complexifiez si nécessaire (U-Net), mais validez toujours spatialement !
